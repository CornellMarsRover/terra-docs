Detailed Description

template<typename T, typename = void>
struct cmr::fabric::IsServerType< T, typename >

A type trait that checks if a type is a server provider which defines a server_t,config_t, andmake_return_ttype along with amake_server()` function.

server_t is the type of the server returned from make_server(). I think of a server as something that runs on a ROS executor, handling requests of some type by invoking user defined callbacks. The name originates from service and action servers, but this definition also incorporates topic subscribers and wall timers.

config_t is the type of the configuration passed to make_server().

make_return_t<T> is a template class that provides a static val member which is the default value of T that will be returned from the server callback if an error occurs or the server is disabled. So if a callback returns T, and an error occurs during that callback or a request is received when the server is inactive but the callback is still active (the latter shouldn't really happen), then it will return make_return_t<T>::val

make_server<T>() will take a reference to the owning LifecycleNode, one or more callback functions for the server to execute, and a config_t object to pass more server specific config parameters. The function should also be a template, templated on the type of the first callback function.

For simplicity, this SFINAE concept does not enforce the make_server member
