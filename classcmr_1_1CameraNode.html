Detailed Description

CameraNode

The camera node reads from a USB camera and outputs the image data as sensor_msgs::Image messages on the "image_raw" topic. It also publishes camera information on the "camera_info" topic using a CameraInfoManager from the ROS2 Perception library.

The configuration options for this node are as follows:

  • "frame_id" -> a string, the frame ID to be passed in the headers of messages published by this node (default: "camera").
  • "image_width" -> an integer, the width of the image in pixels (default: 1280)
  • "image_height" -> an integer, the height of the image in pixels (default: 720)
  • "fps" -> a double, the number of frames per second to take from the camera (default: 10.0)

Under the hood, this node uses the OpenCV image capturing library to read from the camera, and then converts this data into sensor_msgs::Image messages before sending them onto the "image_raw" topic. For camera information, see the online documentation for the CameraInfoManager provided by ROS2 Perception.
